# Angular + Cognito configuration

The Angular app gets its API URL and Cognito settings at **runtime** from `/config.json`, generated by `deploy/scripts/fetch-stack-config.js`. Users are created manually in the Cognito User Pool (see [README – Manual user creation](./README.md#manual-user-creation-cognito)).

## 1. Stack outputs you need

After `sam deploy`, get:

- **ApiUrl** – API Gateway base URL (e.g. `https://xxxxxxxx.execute-api.us-east-1.amazonaws.com/prod`)
- **UserPoolId** – Cognito User Pool ID
- **UserPoolClientId** – Cognito App Client ID
- **AWS Region** – same region as the stack (e.g. `us-east-1`)

## 2. How config reaches the frontend

**Do not hand-edit `environment.prod.ts`** — it contains empty fallback values only.

The config flow is:

1. `sam deploy` creates the stack with outputs (ApiUrl, UserPoolId, UserPoolClientId)
2. `node deploy/scripts/fetch-stack-config.js <stack-name> [region]` reads stack outputs and writes `frontend/public/config.json`
3. `ng build` copies `public/config.json` into the dist output (via `angular.json` assets)
4. The built frontend is deployed to S3 (`aws s3 sync`)
5. At runtime, `AppConfigService` (via `APP_INITIALIZER`) fetches `/config.json` and uses those values

If `/config.json` is missing or fails to load, the app falls back to `environment.prod.ts` (empty values) and logs a console error.

### Quick deploy

```bash
./deploy/scripts/deploy-frontend.sh <stack-name> <s3-bucket> [region]
```

See [README – Deploy the Angular frontend](./README.md#3-deploy-the-angular-frontend) for details.

## 3. Using Cognito in the app

- Use **Amazon Cognito Identity SDK** (e.g. `amazon-cognito-identity-js`) or **AWS Amplify Auth** to sign in and obtain the JWT.
- Send the JWT on API requests in the `Authorization` header (e.g. `Authorization: Bearer <idToken>`).
- API Gateway's Cognito authorizer will validate the token for routes that use `CognitoAuthorizer`.

`ApiService` reads the API URL from `AppConfigService.apiUrl`. The `authInterceptor` should attach the Cognito ID token to outgoing requests.

## 4. Development

Keep `frontend/src/environments/environment.ts` for local dev (e.g. `apiUrl: 'http://localhost:3000'`). Add a `cognito` block there too if you use Cognito in dev (e.g. a separate dev User Pool or mock).

You can also place a `config.json` in `frontend/public/` for local testing with a real stack — the runtime loader will pick it up via `ng serve` too.
